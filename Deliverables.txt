
1. List of Used Tools and Configuration Changes
Tools Used:

    iw – Manage and configure wireless interfaces.
    wpa_supplicant – Connect to Wi-Fi networks using WPA/WPA2/WPA3 security.
    wpa_cli – Interact with wpa_supplicant in real-time.
    tcpdump / Wireshark – Capture and analyze network traffic.
    horst / wavemon – Monitor signal strength, noise, and interference.
    lspci - Provides detailed information about each connected PCI device, including vendor, device ID, and description

6. Wi-Fi captures with the full connection flow between the Wi-Fi client and secured Wi-Fi AP. Be ready to explain Wi-Fi connection flow (including 4-way handshake) and different types of Wi-Fi frames (data, control, management frames).

To decrypt WPA2/WPA3 traffic, you need to capture the 4-Way Handshake, which is exchanged between the client and access point during the connection process.

Wi-Fi Connection Flow (Including 4-Way Handshake):

Step 1: Probe Request / Probe Response

Frame Type: Management
Details: The client sends a Probe Request frame to search for available networks. The AP responds with a Probe Response frame containing details about the network.

Step 2: Authentication

Frame Type: Management
Details: The client sends an Authentication Request to the AP, and the AP responds with an Authentication Response.
This step may involve Open System Authentication or Shared Key Authentication, depending on the security configuration.

Step 3: Association Request / Association Response

Frame Type: Management
Details: The client sends an Association Request to the AP, and the AP responds with an Association Response to confirm the connection parameters.

Step 4: 4-Way Handshake

Frame Type: Management
The 4-Way Handshake is a key exchange process that ensures both the AP and client can securely communicate by verifying the session keys.
Handshake Frames:

Message 1: The AP sends a Message 1 to the client, containing a nonce (random number).
Message 2: The client responds with Message 2, which includes its own nonce and the MIC (Message Integrity Code).
Message 3: The AP sends Message 3, confirming the keys and the session.
Message 4: The client acknowledges the handshake by sending Message 4 back to the AP.

Step 5: Data Transfer

Frame Type: Data
Details: Once the 4-Way Handshake is complete, both the AP and the client can start exchanging Data frames, which carry actual user data (e.g., packets from a web page, video streaming, etc.).

Step 6: Deauthentication / Disassociation (if applicable)

Frame Type: Management
Details: If the client or AP decides to terminate the connection, either can send a Deauthentication or Disassociation frame to end the session.

Types of Wi-Fi Frames:

Wi-Fi communication uses three major types of frames:

Management Frames: Used to establish, maintain, and terminate a connection. Examples include Probe Requests/Responses, Authentication, Association, and Deauthentication frames.

Control Frames: Used to control the flow of data between devices. For example, RTS (Request to Send), CTS (Clear to Send), and ACK (Acknowledgment) frames.

Data Frames: Used to transmit the actual user data (e.g., packets from web traffic or other services).
